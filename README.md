# 🚀 [온콜]

> 핵심 요약: 서비스 장애를 최소화하고 사용자의 불편을 줄이기 위해 미리 비상근무표를 만드는데 이를 자동화할 수 있는 시스템

---

## 📝 기능 구현 목록

### 1️⃣ 입력 처리 (InputView)

- [ ]  **사용자 입력 받기**: `camp.nextstep.edu.missionutils.Console.readLine()` 활용
- [ ]  **입력 값 검증 (Validation)**:
    - [ ]  공백 또는 빈 값 입력 여부 확인
    - [ ]  데이터 타입(숫자, 문자열 등) 일치 여부 확인
    - [ ]  비즈니스 도메인 규칙(범위, 중복, 특정 포맷) 확인
- [ ]  **재시도 로직**: 예외 발생 시 `[ERROR]` 메시지 출력 후 해당 부분부터 재입력 유도

### 2️⃣ 도메인 로직 (Domain/Business Logic)

- [ ]  **데이터 모델링**: 요구사항의 핵심 객체 설계
- [ ]  **핵심 계산 로직**:
    - [ ]  데이터 필터링 및 정렬
    - [ ]  산술 계산 (총합, 평균, 비율 등)
- [ ]  **상태 판별**: 입력값에 따른 등수, 등급, 성공 여부 등의 비즈니스 상태 결정
- [ ]  **Enum 활용**: 상수 데이터 및 관련 로직 관리

### 3️⃣ 결과 출력 (OutputView)

- [ ]  **중간 상태 출력**: 사용자가 진행 상황을 알 수 있는 목록이나 상태 출력
- [ ]  **최종 결과 포맷팅**:
    - [ ]  숫자 천 단위 콤마(`,`) 처리
    - [ ]  날짜 및 시간 포맷팅
    - [ ]  요구사항에 명시된 구분선(`--`) 및 안내 문구 적용
- [ ]  **수익률/통계 출력**: 최종 계산 결과 및 통계 요약 정보 출력

## 🏗️ 설계 원칙 및 패키지 구조

### 📦 패키지 구조

- `controller`: 프로그램의 전체 흐름(Input → Domain → Output)을 제어
- `domain`: 핵심 비즈니스 로직과 상태를 관리 (UI 의존성 없음)
- `view`: 사용자와의 인터페이스 담당 (InputView, OutputView)
- `util/validator`: 공통
- `constant`: 에러 메시지, 공통 상수 등 관리

### 🛠️ 프로그래밍 요구사항 준수

- [ ]  **함수 분리**: 한 함수가 한 가지 기능만 담당하도록 15라인 이내 구현
- [ ]  **Depth 제한**: `indent depth`를 2단계 이하로 유지
- [ ]  **Clean Code**: `else` 예약어를 지양하고 `Early Return` 패턴 적용
- [ ]  **테스트**: 도메인 로직의 핵심 기능에 대한 단위 테스트 작성

---

## 🚨 예외 처리 규칙

- 모든 예외 메시지는 **`[ERROR]`*로 시작해야 함
- 상황에 맞는 예외 타입(`IllegalArgumentException`, `IllegalStateException`) 발생 시킴
- 예외 발생 지점부터 재입력을 받는 흐름을 `Controller`에서 제어

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링 (기능 변화 없음)
- `docs`: README 등 문서 수정
